三步建立自己的博客

发表于 2017-03-19 | | 阅读次数

心里有无数idea在飘无法表达？学了新技能成就满满想装逼却无人知道？想搭博客又被高昂的服务器费用吓退？看完这篇文章，只需要三步，手把手教你免费拥有自己的博客。

1. 拥有github账号和安装git

这一步比较简单，这里就不详细叙述，github账号直接官网注册，git直接下载后一路安装就可以。实在不懂可以参考我的CSDN博客里的文章。

2. 安装node.js和Hexo

打开node.js的官网下载node.js安装包（打开页面后如图1），虽然推荐大部分用户使用V6.10.0版本，但是谁让我喜欢尝鲜呢？于是我下载了V7.7.1版本的。下载过程比较慢，经常有时下着下着就断了，不知道是因为我网速慢还是被墙的原因，总之我下载这个几M的安装包花了我差不多两个小时。
下载完成后直接打开，下一步安装就可以了。

图 1. 下载node.js

安装好Node.js后，打开gitBash，在里面输入npm install -g hexo-cli命令即可安装hexo，安装成功后最末尾会出highlight_alias.json英文字样（如图2）。

图 2. 安装hexo成功

3. 部署博客

  • 创建博客文件夹
    成功安装后，进入你喜欢的文件地方（这里以F盘为例），新建一个文件夹，如githubBlog，然后双击进入，右键选择git bash here打开git bash，然后输入命令hexo init blog或hexo init（前者blog为博客生成特定名称，若不加则会使用默认），之后git会自动下载相应的文件及默认的博客模板，完成后会出现start blogging with Hexo英文字样，如图3：

    图 3. 初始化博客成功

    成功提示后，新建的文件夹下会自动生成blog文件夹，如图4所示：

图 4. 生成的博客截图

生成的博客文件夹下有个_config.yml文件，用来专门配置博客的一些属性，属性的作用如下所示：


#Hexo Configuration
##Docs: https://hexo.io/docs/configuration.html
##Source: https://github.com/hexojs/hexo/
#Site
title: #标题
subtitle: #副标题
description: #站点描述，给搜索引擎看的
author: 
email: 邮箱
language: zh-CN
timezone: Asia/ShangHai
#URL
##If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://mhwang7758
root: /
permalink: :year/:month/:day/:title/ #文章的链接格式
permalink_defaults:
#Directory
source_dir: source     
public_dir: public #生成的网页文件目录
tag_dir: tags  #标签目录
archive_dir: archives  #存档目录
category_dir: categories #分类目录
code_dir: downloads/code
i18n_dir: :lang
skip_render:
#Writing
new_post_name: :title.md # File name of new posts #新文章标题
default_layout: post #默认的模板，包括 post、page、photo、draft（文章、页面、照片、草稿）
titlecase: false # Transform title into titlecase #标题转换成大写
external_link: true # Open external links in new tab #在新选项卡中打开连接
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
future: true
highlight:
  enable: true
  line_number: true #显示行号
  auto_detect: false
  tab_replace:
#Category & Tag
default_category: uncategorized
category_map:
tag_map:
#Date / Time format
##Hexo uses Moment.js to parse and display date
##You can customize the date format as defined in
##http://momentjs.com/docs/#/displaying/format/
date_format: YYYY-MM-DD
time_format: HH:mm:ss
#Pagination
##Set per_page to 0 to disable pagination
per_page: 10
pagination_dir: page
#Extensions
##Plugins: https://hexo.io/plugins/
##Themes: https://hexo.io/themes/
theme: next #主题
#Deployment
##Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: 创建的git仓库地址
  branch: master

  • 查看博客初形
    按照网上的很多教程，完成这一步后输入hexo generate和hexo server这两个命令后就可以打开http://localhost:4000看网页了。然而事实是残酷的，当输入hexo server回车后会提示没有这个命令！
    这是因为由于版本问题，Hexo 3.0 后server被单独出来了，需要安装server，命令为npm install hexo -server --save（注意需要在博客目录下打开git bash），如图5：

    图 5. 安装server

    看见有start processing后之后 就可以打http://localhost:4000查看博客初形了，如图6：

    图 6. 博客初形

  • 将博客与github page绑定
    接着登陆github官网，点右上角的create new respository，在repository name一栏填上你的github名+“.”+github.io，如xxx.github.io（注意你的名字与github之间有“.”）。如图7：

    图 7. 创建Repository

    一切就绪后，按照网上的一些教程，文章写好后直接hexo deploy即可，但是会发现又找不到命令。但是如果你上网搜，大部分答案会告诉你输入npm install hexo-deployer-git --save命令，或者将里的type中的github改成git，最后你都试了还是然并卵，最新的版本应该是输入npm install --save hexo-deployer-git（–save到前面来了）。

如果到了这里，恭喜你你已经创建了自己的个人博客，接下来请尽情喷洒你的灵感与创意吧！
